The workflow

PyQuiz is built so that creating a week's worth of activities — and editing them — needn't take more than an hour. The shape of the work is always the same:

  1. Describe what you want. Decide the year group, topic, lesson length and how you'd like it structured.
  2. Generate the JSON. Paste the PyQuiz prompt into ChatGPT, Claude, Gemini or Copilot, then add your request. The AI returns a complete pack.
  3. Import it. Open the authoring tool, choose Import and drop in the JSON (or a .pyquiz file).
  4. Fine-tune. Reorder, adjust difficulty, sharpen wording, fix anything the AI got wrong. Preview each activity exactly as students see it.
  5. Share. Export as a .pyquiz file and add it to your packs folder, or hand students the file directly.
Why AI first? The prompt below teaches the model the full set of activity types, the exact data format and the teaching principles (fading support, sensible difficulty, sectioning). That means it returns something genuinely usable — a first draft you refine, not a blank page you fight.

Writing your request

After the prompt, describe what you want in plain English. Be specific about the things that matter — year group, topic, length, and any particular structure or counts. For example:

Create a 30-minute activity on selection for ages 12–13 with 5 final
challenges. Chunk it into easy-to-follow sections with a logical
progression.

Other things worth stating: a misconception to target ("students confuse = and =="), a context ("use a login-attempts example"), or an exam focus ("advanced level, emphasise boundary testing"). The more specific the brief, the closer the first draft.

Activity types

A pack mixes eight activity types. Variety lets you drill one skill from several angles; the prompt knows how to choose and sequence them.

TypeWhat the student does
Predict outputReads code and predicts exactly what it prints (free text or multiple choice).
ParsonsReorders shuffled lines — and bins distractor lines — into a working program.
ClozeFills gaps in code: free text, a drop-down, or a drag-from word bank.
Trace tableSteps through code line by line, recording how each variable changes.
FlowchartCompletes the missing labels in a flowchart of the algorithm.
Spot the bugIdentifies the broken line and fixes it.
ModifyChanges working code to meet a new requirement.
TestingDesigns test data — normal, boundary, invalid and erroneous — for a snippet of code. The teacher leaves cells blank for the student to complete and the value is checked automatically against the input's type and range.

(There's also a ninth, starter challenge: an open "now build it" task with a worked example and a model solution, used to close a section.)

A taste of the format

You rarely need to read the JSON — the tool and the prompt handle it — but here's a single predict_output activity so you can see the shape:

{
  "type": "predict_output",
  "title": "Counting up",
  "instructions": "Predict exactly what this code prints.",
  "difficulty": 1,
  "renderer_version": 1,
  "payload": {
    "direction": "code_to_output",
    "mode": "free_text",
    "code": "for i in range(4):\n    print(i)",
    "answer": "0\n1\n2\n3",
    "accepted_answers": ["0\n1\n2\n3"]
  }
}

Fine-tuning in the tool

This is where your judgement lands. In the authoring tool you can:

  • Reorder activities by dragging, and manage sections.
  • Adjust each activity's difficulty on a five-point slider.
  • Edit code, wording, hints and feedback, with a live preview of the student's view.
  • Add or delete activities, and tidy flowcharts (the editor auto-arranges routing, or you can hand-route).

Validation runs continuously in the background — if something's structurally wrong, the tool flags it as you work, so there's no separate "check" step.

Sharing packs

Use Export and download the .pyquiz file. You can hand it straight to students (they load it on the student page), or add it to your site so it appears in the chooser:

  1. Drop the file in packs/ in your repository.
  2. Add one entry to packs/index.json — the Export dialog gives you the exact snippet to paste.
  3. Commit and push. The student chooser reads the manifest, so the new pack appears automatically, grouped by level.

Renderer versions

Every activity is stamped with a renderer_version. If a future PyQuiz update changes how an activity type works, old packs keep rendering with the version they were built against — so packs you make today won't break later. You don't need to manage this; the tool stamps it for you.
